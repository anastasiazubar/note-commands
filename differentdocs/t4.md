## Test 4 – Unauthorized Activation Group Membership Validation

### Objective

Ensure users are not directly assigned to Activation Groups unless they have an active TEALAS approval.

### Validation

1. Identify all Activation Groups.

2. Retrieve current members of each Activation Group from GCP / Cloud Identity.

3. For each Activation Group member, verify there is a valid active TEALAS approval record.

4. Confirm the TEALAS approval is for the same user and corresponding privileged access request.

5. Verify the TEALAS approval period has not expired.

6. Flag any Activation Group member who does not have a matching active TEALAS approval.

### Failure Condition

A user is a member of an Activation Group without a valid active TEALAS approval.

### Alert

Critical severity email notification sent to IAM Operations and TEALAS Support teams including:

* Activation Group Name
* User ID / Email
* Missing or Invalid TEALAS Approval Status
* Detection Timestamp
